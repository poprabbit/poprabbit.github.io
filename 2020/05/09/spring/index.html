  2. // 我们把这步叫做 注册Bean 吧

  4. Bean 创建、装填属性、回调 各种初始化函数(init-method,InitializingBean,BeanPostProcessor)

aop

spring 1.2 支持的 aop 配置

  1. 配置bean
  2. 配置Advice –指定执行前/执行后，处理逻辑
  3. 指定Advisor–配置指定方法用指定Advice增强 4、配置Proxy–配置指定的bean用指定的Advisor增强

缺点：我们得为每个 bean 都配置一个代理，之后获取 bean 的时候需要获取这个代理类的 bean 实例（如 (UserService) context.getBean(“userServiceProxy”)）

autoproxy 方式

  1. ….
  2. 配置 RegexpMethodPointcutAdvisor
  3. 配置 DefaultAdvisorAutoProxyCreator

Advisor 有一个更加灵活的实现类 RegexpMethodPointcutAdvisor，它能实现正则匹配 指定方法增强
DefaultAdvisorAutoProxyCreator 让 ioc 容器中的所有 advisor 来匹配方法，advisor 内部都是有 advice 的，让它们内部的 advice 来执行拦截处理。

Spring 2.0 @AspectJ 配置

  1. 开启 @AspectJ 的注解配置方式
  2. 配置 Pointcut 用于定义哪些方法需要被增强或者说需要被拦截
  3. 配置 Advice ，JoinPoint用于获取方法入参

Pointcut 中可以使用 execution within annotation 等方式配置

  • Spring AOP 的原理很简单，就是动态代理，它和 AspectJ 不一样，AspectJ 是直接修改掉你的字节码。
  • 如果被代理的目标类实现了一个或多个自定义的接口，那么就会使用 JDK 动态代理，如果没有实现任何接口，会使用 CGLIB 实现代理，如果设置了 proxy-target-class=”true”，那么都会使用 CGLIB。
  • InvocationHandler 只有一个方法，当生成的代理类对外提供服务的时候，都会导到这个方法中

很多人都以为 Spring AOP 是通过这个接口来作用于 bean 生成代理的。 它和 BeanPostProcessor 的方法非常相似，而且它还继承了 BeanPostProcessor。
InstantiationAwareBeanPostProcessor 是 Instantiation，BeanPostProcessor 是 Initialization 也有创建代理的逻辑，需要配置一个 customTargetSourceCreators。
